## 📅 2026-06-29: Supabase 로그인 및 모바일 반응형 UI 최적화

### 1. Supabase OAuth 로그인 (Google) 리다이렉트 문제 해결
- **문제 현상:** 구글 로그인 시 모바일 기기(내부망 IP: 192.168.x.x)에서 로그인 완료 후 앱으로 돌아오지 못하고 `localhost`로 튕기거나 "사이트에 연결할 수 없음" 에러가 발생하는 현상.
- **원인:** Supabase는 등록된 Redirect URL과 100% 일치하지 않으면 무조건 기본 설정된 'Site URL'(`localhost:3000`)로 강제 리다이렉트하는 깐깐한 보안 정책을 가지고 있음.
- **해결책 (영구적 세팅 방법):**
  1. **Site URL:** `https://sf-farm.vercel.app` (실제 배포된 정식 주소로 고정)
  2. **Redirect URLs (추가 허락 주소):** 
     - `http://localhost:3000/*` (로컬 PC 작업용)
     - `http://192.168.219.145:3000/*` (모바일/내부망 테스트용)
  - **주의:** 주소 끝에 와일드카드(`*`)를 반드시 붙여야 경로(ex. `/dashboard`)가 동적으로 붙어도 모두 통과됩니다. 또한 설정 후 캐시가 갱신되기까지 약 2~3분이 소요됩니다.

### 2. 구글 시트 iframe 모바일 스크롤 버그 해결
- **문제 현상:** 핸드폰에서 구글 시트를 `iframe`으로 불러올 때, 처음 1~2행만 보이고 스크롤이 먹통이 되어 값을 입력할 수 없는 문제.
- **원인:** 모바일 브라우저(특히 iOS Safari)는 `flex-1` 안에 갇힌 `iframe`의 내부 스크롤 엔진을 제대로 계산하지 못하고 프레임을 내용물의 끝부분까지 무한대로 강제 확장시켜 버리는 고질적인 렌더링 버그가 있습니다. 이로 인해 스크롤이 부모 뷰로 넘어가거나 잘리게 됩니다.
- **해결책 (UX 최적화):**
  - **PC/태블릿:** 기존처럼 넓은 화면을 활용하여 `iframe` 내에서 직접 편집.
  - **모바일 환경 (반응형 대응):** 모바일 웹 에디터는 `iframe` 환경에서 극도로 사용성이 떨어지므로, Tailwind의 `md:hidden` 클래스를 활용해 `iframe`을 숨깁니다. 대신 **[Open Google Sheets]** 전용 버튼 카드를 띄워 구글 시트 앱(또는 새 창 전체화면)으로 쾌적하게 넘어가도록 설계하는 것이 최신의 모바일 UX 표준입니다.

### 3. 모바일 리스트 UI 레이아웃 가독성 최적화 (가로 한 줄 배치)
- **문제 현상:** PC에서는 가로 한 줄에 4개의 정보(체크박스, 날짜, 내용, 휴지통)가 예쁘게 나열되나, 모바일에서는 공간 부족으로 세로로 4줄이 쌓여 공간 낭비 및 가독성 저하 발생.
- **해결책:**
  - `flex-col`로 쌓이게 두지 않고 `flex-row items-start` 구조를 강제 유지.
  - 모바일에서는 폰트 크기(`text-xs`), 여백(`px-2 py-1.5`), 아이콘 크기를 미세하게 줄여 1줄 안에 우겨넣어도 어색하지 않은 '콤팩트 뷰'로 전환.
  - 제목이나 내용이 길어지는 것에 대비해 남은 공간(`flex-1 min-w-0`)을 할당하고, 말줄임표 처리(`truncate`, `line-clamp-1`)로 텍스트 폭주 방지.

### 4. 텍스트 말줄임표(...) 확장 툴팁 방식의 UX 선택
- **질문:** 내용이 잘린 부분을 클릭하면 '말풍선(툴팁)'으로 전체 내용을 보여줄 수 있는지?
- **가이드:** 모바일에서 말풍선(Tooltip)은 다른 버튼을 가리거나 터치 후 끄기가 어려워 좋은 경험을 주지 못합니다.
- **해결책 (아코디언 인라인 확장):** 
  - 클릭(터치) 시 `expandedEventId` 상태를 토글하여, 리스트 내에서 글자가 **아래로 부드럽게 펼쳐지는 (Accordion) 방식**을 적용했습니다. 이는 최신 모바일 앱 리스트(ex. 메일 앱, 알림 센터)에서 사용하는 표준 방식입니다.
