## 📅 2026-06-29: Supabase 로그인 및 모바일 반응형 UI 최적화

### 1. Supabase OAuth 로그인 (Google) 리다이렉트 문제 해결
- **문제 현상:** 구글 로그인 시 모바일 기기(내부망 IP: 192.168.x.x)에서 로그인 완료 후 앱으로 돌아오지 못하고 `localhost`로 튕기거나 "사이트에 연결할 수 없음" 에러가 발생하는 현상.
- **원인:** Supabase는 등록된 Redirect URL과 100% 일치하지 않으면 무조건 기본 설정된 'Site URL'(`localhost:3000`)로 강제 리다이렉트하는 깐깐한 보안 정책을 가지고 있음.
- **해결책 (영구적 세팅 방법):**
  1. **Site URL:** `https://sf-farm.vercel.app` (실제 배포된 정식 주소로 고정)
  2. **Redirect URLs (추가 허락 주소):** 
     - `http://localhost:3000/*` (로컬 PC 작업용)
     - `http://192.168.219.145:3000/*` (모바일/내부망 테스트용)
  - **주의:** 주소 끝에 와일드카드(`*`)를 반드시 붙여야 경로(ex. `/dashboard`)가 동적으로 붙어도 모두 통과됩니다. 또한 설정 후 캐시가 갱신되기까지 약 2~3분이 소요됩니다.

### 2. 구글 시트 iframe 모바일 스크롤 버그 해결
- **문제 현상:** 핸드폰에서 구글 시트를 `iframe`으로 불러올 때, 처음 1~2행만 보이고 스크롤이 먹통이 되어 값을 입력할 수 없는 문제.
- **원인:** 모바일 브라우저(특히 iOS Safari)는 `flex-1` 안에 갇힌 `iframe`의 내부 스크롤 엔진을 제대로 계산하지 못하고 프레임을 내용물의 끝부분까지 무한대로 강제 확장시켜 버리는 고질적인 렌더링 버그가 있습니다. 이로 인해 스크롤이 부모 뷰로 넘어가거나 잘리게 됩니다.
- **해결책 (UX 최적화):**
  - **PC/태블릿:** 기존처럼 넓은 화면을 활용하여 `iframe` 내에서 직접 편집.
  - **모바일 환경 (반응형 대응):** 모바일 웹 에디터는 `iframe` 환경에서 극도로 사용성이 떨어지므로, Tailwind의 `md:hidden` 클래스를 활용해 `iframe`을 숨깁니다. 대신 **[Open Google Sheets]** 전용 버튼 카드를 띄워 구글 시트 앱(또는 새 창 전체화면)으로 쾌적하게 넘어가도록 설계하는 것이 최신의 모바일 UX 표준입니다.

### 3. 모바일 리스트 UI 레이아웃 가독성 최적화 (가로 한 줄 배치)
- **문제 현상:** PC에서는 가로 한 줄에 4개의 정보(체크박스, 날짜, 내용, 휴지통)가 예쁘게 나열되나, 모바일에서는 공간 부족으로 세로로 4줄이 쌓여 공간 낭비 및 가독성 저하 발생.
- **해결책:**
  - `flex-col`로 쌓이게 두지 않고 `flex-row items-start` 구조를 강제 유지.
  - 모바일에서는 폰트 크기(`text-xs`), 여백(`px-2 py-1.5`), 아이콘 크기를 미세하게 줄여 1줄 안에 우겨넣어도 어색하지 않은 '콤팩트 뷰'로 전환.
  - 제목이나 내용이 길어지는 것에 대비해 남은 공간(`flex-1 min-w-0`)을 할당하고, 말줄임표 처리(`truncate`, `line-clamp-1`)로 텍스트 폭주 방지.

### 4. 텍스트 말줄임표(...) 확장 툴팁 방식의 UX 선택
- **질문:** 내용이 잘린 부분을 클릭하면 '말풍선(툴팁)'으로 전체 내용을 보여줄 수 있는지?
- **가이드:** 모바일에서 말풍선(Tooltip)은 다른 버튼을 가리거나 터치 후 끄기가 어려워 좋은 경험을 주지 못합니다.
- **해결책 (아코디언 인라인 확장):** 
  - 클릭(터치) 시 `expandedEventId` 상태를 토글하여, 리스트 내에서 글자가 **아래로 부드럽게 펼쳐지는 (Accordion) 방식**을 적용했습니다. 이는 최신 모바일 앱 리스트(ex. 메일 앱, 알림 센터)에서 사용하는 표준 방식입니다.

## 📅 2026-06-30: Supabase 설정 및 프로젝트 혼동 문제 해결

### 1. Supabase 대시보드 설정 변경이 적용되지 않는(유저가 보이지 않는) 문제
- **문제 현상:** `.env.local`의 설정에 맞추어 Supabase 대시보드에서 Redirect URL 설정을 변경했으나 로그인 리다이렉트 문제가 해결되지 않고, `Authentication -> Users` 목록에도 어제 가입한 유저가 보이지 않는 현상.
- **원인:** Supabase에서 여러 개의 프로젝트를 생성했을 경우, **현재 작업 중인 소스코드(`.env.local`의 `NEXT_PUBLIC_SUPABASE_URL`)에 연결된 실제 프로젝트**가 아닌 **실수로 생성된 다른 프로젝트의 대시보드**를 띄워놓고 설정을 변경하고 있었음.
- **해결책 (검증 팁):** 
  - 문제가 발생하면 가장 먼저 `.env.local` 파일의 `NEXT_PUBLIC_SUPABASE_URL` 주소에서 **프로젝트 ID**(예: `uoeridilnkiispxhdfhf`)를 확인합니다.
  - 브라우저 주소창의 Supabase 대시보드 접속 URL(`https://supabase.com/dashboard/project/[여기에 프로젝트 ID]`)과 정확히 일치하는 프로젝트를 보고 있는지 반드시 대조해 보는 습관이 필요합니다.

### 2. IoT 장비 제어 아키텍처 (수동 제어 vs 스케줄 제어)
- **배경:** 'Equipment Control' 화면에서 스케줄(on/off 시간)을 설정할 때마다 아두이노 스케치 코드를 재생성하여 업로드하는 방식은 실무 환경에서 불가능하며, 매우 비효율적입니다.
- **올바른 아키텍처 방향 (Smart Server, Dumb Node):**
  1. **아두이노의 역할 (Dumb Node):** 시간이나 스케줄 개념을 일절 가지지 않습니다. 오직 `smartfarm/pooh/{deviceName}/set` MQTT 토픽을 항상 구독(Subscribe)하고 있다가, `ON` 메시지가 오면 릴레이를 켜고 `OFF` 메시지가 오면 끄는 단순한 역할만 수행합니다.
  2. **수동 제어 (Web UI):** 사용자가 웹 대시보드에서 토글 스위치를 누르면, 웹 브라우저가 직접 MQTT 브로커를 통해 `smartfarm/pooh/{deviceName}/set` 으로 `ON/OFF` 메시지를 보냅니다.
  3. **스케줄 제어 (Smart Controller):** 스케줄 시간에 맞춰 장비가 동작하도록 관리하는 두뇌는 24시간 켜져 있는 서버(예: 라즈베리파이의 파이썬 스크립트)가 담당합니다. 파이썬 스크립트가 1분마다 데이터베이스의 스케줄 세팅값을 확인하고, 켜져야 할 시간이면 알아서 MQTT로 `ON`을 쏘고, 꺼져야 할 시간이면 `OFF`를 쏩니다. (브라우저가 꺼져 있어도 작동 보장)

### 3. MQTT Payload 불일치 및 아두이노 스케치 생성 규칙 강화
- **문제 현상:** 아두이노 스케치는 제어 명령으로 단순 문자열 `"ON"`/`"OFF"`를 기대하도록 작성(생성)되었으나, 웹 UI에서는 `{"equipment":"growLight", "state":"ON"}` 형태의 JSON을 전송하여 하드웨어가 무반응이거나, 혹은 반대로 아두이노 코드가 쓸데없이 `deserializeJson()`을 수행하려다 `InvalidInput` 에러를 뱉는 문제가 발생함.
- **해결책 및 규칙 강화:**
  1. **웹 코드 통일:** `DashboardClient.tsx`와 `client.ts`에서 장비 제어 시 JSON 객체가 아닌 단순히 `"ON"` / `"OFF"` 문자열만 payload로 쏘도록 일치시킴.
  2. **프롬프트 강제화:** AI 모델(특히 파라미터가 적은 Flash 모델 등)이 코드를 생성할 때 혼동하여 `deserializeJson()`을 추가하지 못하도록, 프롬프트의 **11번 규칙(Rule 11)**에 "절대 JSON으로 파싱하려 하지 말고 `message == "ON"`과 같이 단순 문자열 비교만 사용할 것"을 매우 강력한 어조로 명시.
  3. **로그 가시성 확보:** AI가 스케치 생성 시, 제어에 성공하면 `=> 장비명 turned ON/OFF` 처럼 명확한 피드백 로그를 `Serial.println()` 하도록 규칙에 추가하여 디버깅을 용이하게 함.

### 4. 불필요한 에이전트(agent.py) 제거 및 UI 개선

3. **코드 및 모달 창:** 에이전트 파이썬 코드를 생성해주던 `handleGenerateAgentCode` 로직 및 팝업 모달창 완전히 제거

### 5. Remote Logger Control 간격 및 UI 디자인 개선
1. **간격 문제 해결:** 부모 컨테이너에 있던 `justify-between` 정렬 속성을 제거하여, 텍스트 설명과 상태 제어 박스 사이의 불필요하게 넓었던 공백을 없애고 바로 아래에 자연스럽게 붙도록 배치했습니다.
2. **UI 디자인 개선:** 기존에 빈 정보가 들어가면서 투박하게 보이던 네모난 박스 대신, **모서리가 부드럽게 둥근(`rounded-2xl`) 모던한 디자인의 전용 토글 박스**로 완전히 교체했습니다. 스위치가 켜졌을 때 맥박처럼 뛰는 초록색 표시등(`animate-pulse`)과 상태 뱃지 등 좀 더 직관적이고 세련된 형태로 다듬었습니다.

### 6. 시설 커스텀 정렬 기능 구현 원리 및 안전성
질문해 주신 내용에 대한 답변: **"이 변경 사항은 매우 안전하며 부작용(Side Effect)이 거의 없습니다."** 

그 이유는 데이터베이스(DB) 테이블의 뼈대(`device_configs`)를 건드리거나 마이그레이션을 하는 복잡한 방식이 **아니기 때문**입니다. 대신 기존에 와이파이, 센서 설정 등 각종 앱 설정을 저장할 때 이미 널리 사용하고 있는 안전한 저장소(`app_settings` 테이블)에 `"sf_facilities_order"`라는 이름표를 달아 **순서 배열(목록)만 저장**하고 프론트엔드 코드에서 가볍게 정렬하도록 구현했습니다. 이 방식은 추후 시스템을 확장하거나 변경할 때도 충돌 없이 유연하게 동작합니다. 

**구현 완료 사항:**
1. **위/아래 화살표(↑/↓) 버튼 추가:** `Facilities & Crops Management` 탭의 시설 목록 가장 우측(Actions 열)에 추가했습니다.
2. **글로벌 정렬 동기화:** 화살표 버튼을 눌러 순서를 바꾸면, 즉시 데이터베이스에 저장되며 **Home 탭의 대형 시설별 요약 카드**와 **최상단의 시설 선택 가로 탭** 등 대시보드의 모든 곳에 동일한 순서가 동기화되어 나타납니다.

### 7. 불필요한 시스템 중복 기능 제거 및 UI 개선
1. **System Settings 내 중복 UI 제거 (WiFi):** 기존에 System Settings 탭과 Facilities(아두이노 설정) 탭에 중복으로 존재하여 사용성 혼란을 주던 구형 'WiFi Setup (For Devices)' 수동 입력 폼을 완전히 삭제했습니다. 이제 라즈베리파이는 'WiFi Scan & Setup'으로, 아두이노는 각 시설의 설정 메뉴에서만 관리하도록 책임이 명확히 분리되었습니다.
2. **System Settings 내 중복 UI 제거 (MQTT):** 마찬가지로 아두이노 설정(Network & MQTT Configuration)과 완벽하게 기능이 중복되고 실제로 사용되지 않던 레거시 코드인 'MQTT Configuration' 패널을 `System Settings`에서 완전히 삭제하여 대시보드 구조를 더욱 직관적이고 깔끔하게 개선했습니다.

### 8. 대시보드 상태 동기화 및 React 18 Strict Mode 버그 해결
1. **기본(Default) 시설 상태값 동기화 오류 해결:** `currentDeviceId` (주소창의 현재 ID 파라미터)가 비어있을 때 발생하는 버그를 수정했습니다. 주소창이 비어있으면 UI에서는 첫 번째 시설을 자동으로 보여주지만, 백그라운드 데이터 요청 로직은 여전히 `null` 값을 기반으로 데이터를 찾고 있어서 발생한 현상입니다.
   - **조치:** Equipment Control, Sensor Settings 탭 등을 비롯한 전체 코드의 데이터 요청 함수들이 주소창 파라미터가 아닌, '실제로 렌더링되기로 결정된 유효한 기기 ID(`resolvedDeviceId`)'를 바라보도록 전부 교체했습니다. 이제 다른 탭을 다녀오지 않아도 처음 진입 시 곧바로 설정 내역이 정상적으로 렌더링됩니다.
2. **Supabase Realtime Channel 충돌 버그 해결:** 개발 모드(Strict Mode)에서 컴포넌트가 눈 깜짝할 새(동일한 밀리초)에 두 번 마운트되면서 `Cannot add postgres_changes callbacks... after subscribe()` 에러가 발생하는 고질적인 문제를 해결했습니다.
   - **조치:** 채널 생성 이름 끝에 단순 타임스탬프(`Date.now()`)뿐만 아니라 무작위 난수(`Math.random()`)까지 덧붙여, `realtime-telemetry-1782803117919-0.847192...` 처럼 1ms의 오차도 없이 이름이 100% 고유하게 생성되도록 보강하여 충돌을 원천 차단했습니다.
